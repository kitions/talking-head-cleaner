# 剪辑决策规则

这份文档描述“什么该删、什么不该删、什么只进入复核清单”。

## 总原则

目标是让口播更干净，但不牺牲语义。

优先级：

1. 不误删有效内容；
2. 清理明显 filler；
3. 保持音画同步；
4. 切点自然，不吞字；
5. 所有自动删除都可追溯。

## 词类分级

### A 类：明确删除

独立出现，且不承载语义的 filler：

```text
嗯
呃
额
唔
呣
```

典型例子：

```text
嗯，我当时就去了医院
呃，然后医生跟我说
额，这个事情是这样的
```

处理方式：

- 自动删除；
- 前后加安全余量；
- 切点做短淡入淡出；
- 记录到 manifest。

### B 类：大概率删除

上下文显示是语气词的“啊”，以及句首卡顿。

典型例子：

```text
啊，我想起来了
然后，啊，医生就说
```

处理方式：

- `aggressive` 模式自动删除；
- `safe` 模式只记录或谨慎删除；
- 如果和后字粘连严重，进入 review。

### C 类：谨慎处理

半语义口头禅：

```text
这个
就是
然后
然后呢
那个
其实
就是说
```

这些词有时啰嗦，有时是自然表达的一部分。

处理方式：

- 当前版本不自动删除；
- 后续 `editor` 模式可以标记高频重复；
- 只有在重复开头或明显重来时才考虑自动删。

### D 类：默认保留

可能承载情绪、强调或句子结构的声音：

```text
哎呀
天哪
啊？ 
啊！
```

例如：

```text
天哪，这个太痛苦了
哎呀，当时真的吓到了
```

这些不是无意义 filler，默认保留。

## 时间戳策略

不要只切 Whisper 给出的精确词区间，因为词级时间戳经常只覆盖声音中间。

建议：

```text
实际切点 = 词时间戳 ± 安全余量
```

当前经验值：

```text
前置余量：70–100ms
后置余量：100–140ms
音频淡入淡出：25–30ms
```

后续更好的方案是：在词时间戳前后 100–200ms 内找音频能量低谷，把切点落在低谷上。

## 合并规则

如果多个切点距离很近，应合并成一个切点。

建议：

```text
gap <= 80ms：合并
gap > 80ms：保留独立切点
```

原因：

- 太近的两个切点会产生连续跳切；
- 合并后听感通常更自然；
- manifest 仍可记录多个原因。

## 长停顿规则

长停顿不一定全部删除，建议缩短。

默认：

```text
超过 1.2s 的无意义停顿 → 缩短到约 0.4s
```

保留：

- 情绪停顿；
- 强调停顿；
- 换气停顿；
- 信息密度高的转场停顿。

## 重复表达规则

重复表达比 filler 更危险，不能只靠文字相等判断。

可自动删除的情况：

```text
我当时就，我当时就去了医院
```

保留或 review 的情况：

```text
我很害怕，真的很害怕
```

第一种是口误重来，第二种可能是强调。

后续 `editor` 模式建议使用：

- 文本相似度；
- 时间间隔；
- 前后语义完整度；
- 第二次表达是否更完整；
- 是否存在明显停顿或重启语气。

## Review-only 规则

以下情况只记录，不自动删：

- 置信度低；
- 时间戳异常长；
- 和有效词粘连；
- 删除后可能导致句子断裂；
- 多模型结果冲突；
- 语气词可能是情绪表达。

Review 清单应包含：

```json
{
  "start": 12.34,
  "end": 12.78,
  "text": "啊",
  "reason": "possible_filler_but_context_sensitive",
  "suggestion": "manual_review"
}
```
